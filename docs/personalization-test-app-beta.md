# Project 1 - Personalization Test App (Beta)

## Overview

The Personalization Test App is an internal tool built for Phera's researchers. Its sole purpose is to evaluate the quality of the AI backend's personalization engine - specifically, whether responses meaningfully adapt based on a user's health profile, demographics, and symptoms.

This is not a user-facing product. It is a controlled evaluation environment where researchers submit test cases and judge response quality by reading the output. There is no rating system, no feedback loop, and no user accounts. It is designed for internal evaluation only.

## Purpose

The core question this app answers is: *does the AI backend actually personalize results accurately?*

A researcher observes a randomly generated pH value, fills in a health profile (age, diagnoses, symptoms, ethnic background, etc.), submits, and reads the response. They evaluate whether the response made sense for that specific profile - whether it acknowledged relevant conditions, cited appropriate research, and gave contextually accurate insights. This is done externally and manually, not inside the app. The app also allows researchers to export results in PDF, JSON, and CSV formats.

Over time, this tool helps the team understand:

- Where the AI responses are strong
- Where personalization is missing or generic
- Which health profiles produce unexpected results
- Whether newly ingested research papers improve response quality

## Who Uses It

- Researchers evaluating AI response quality
- Phera internal team only - URL is not shared publicly

## End-to-End Flow

```text
Researcher
  |
  v
Frontend
  |  App randomly generates a pH value (displayed to researcher)
  |  Researcher fills in optional health context:
  |    age, diagnoses, symptoms, ethnic background,
  |    birth control, hormone therapy, fertility journey
  |  Clicks Submit
  |
  v
BFF - Beta Mode
  |  No authentication
  |  Forwards request as-is to AI backend
  |
  v
AI Backend - Beta Instance
  |
  |-- Builds health profile from submitted form fields
  |
  |-- Calls GPT-4o-mini -> converts structured form data
  |   into a natural language medical research query
  |
  |-- LangGraph pipeline runs:
  |
  |   Retrieve Node:
  |     -> Hybrid search (pgVector semantic + BM25 keyword)
  |     -> Cross-encoder reranking (ms-marco-MiniLM-L-6-v2)
  |     -> Returns top-ranked research paper chunks
  |
  |   Reasoning Node (GPT-4o):
  |     -> Analyzes retrieved chunks against health profile
  |     -> Identifies most clinically relevant evidence
  |     -> Structures reasoning before response generation
  |
  |   Generate Node (GPT-4o):
  |     -> Produces personalized response with inline citations
  |     -> Disclaimer embedded within the response text
  |
  |-- Writes anonymous QueryLog to database
  |   (pH value, generated query, response, citations, timing)
  |   user_id = null - no identity attached
  |
  '-- Returns response to frontend
        agent_reply (includes disclaimer), citations[]

  v
Researcher reads the response and evaluates quality
```

## What the Frontend Displays

- The personalized response text (`agent_reply`) - disclaimer is embedded within this
- Citations - paper titles and relevant excerpts (`citations[]`)

## Data Storage

All data storage is handled entirely on the AI backend side. The frontend and BFF store nothing.

| What | Stored | Details |
|---|---|---|
| QueryLog | Yes | Anonymous row - pH value, generated query, full response, citations, processing time. `user_id = null`. For debugging only. |
| User identity | No | No accounts, no login, no user records |
| Health profile | No | Form data is used for this query only, not saved |

## What This App Does Not Do

- No user authentication or login
- No persistent user accounts or health profiles
- Not accessible to the public - internal URL only

## Integration Notes

**BFF:**

- Run in `beta` mode for this product
- No `Authorization` header - do not send one
- No `X-User-Id` header - do not send one
- Forward the request body as-is to the AI backend
- AI base URL will be provided after deployment

**Frontend:**

- App randomly generates a pH value and displays it to the researcher
- Researcher fills in optional health context fields
- One submission -> one response
- Display: `agent_reply` and `citations[]` only
- No login screen, no session management, no history view

**CV:**

- Not applicable for this project. CV is not involved in the Beta app. pH value is randomly generated by the frontend.

## Deployment

## Technical Stack (AI Backend)

| Component | Technology |
|---|---|
| API framework | FastAPI (Python) |
| Agent orchestration | LangGraph |
| Retrieval | LlamaIndex + pgVector (hybrid search) |
| Reranking | sentence-transformers (ms-marco-MiniLM-L-6-v2) |
| Reasoning | Azure OpenAI GPT-4o |
| LLM (generation) | Azure OpenAI GPT-4o |
| Query generation | Azure OpenAI GPT-4o-mini |
| Vector database | PostgreSQL + pgVector (GCP Cloud SQL) |
| Research paper storage | GCP Cloud Storage bucket |
